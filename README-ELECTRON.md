# GeoRail — сборка десктопного приложения (Electron)

Это приложение теперь можно запускать как обычную программу с окном — без терминала.
Внутри установлен Node.js и сам сервер расписания; пользователю достаточно скачать один установщик.

---

## Что изменилось в проекте

- Добавлена папка `electron/` — обёртка: главный процесс (`main.js`), заставка (`loading.html`), preload.
- `config.js` — каталог данных теперь берётся из переменной `MTR_DATA_DIR` (в собранном приложении это
  пользовательская папка `%APPDATA%\GeoRail\data`), а порт может быть `0` (свободный).
- `server.js` — сервер экспортируется и сообщает реальный порт; обычный запуск `node server.js` по-прежнему работает.
- `package.json` — добавлены зависимости Electron и конфиг сборщика `electron-builder`.
- `build/` — иконки приложения (`icon.ico`, `icon.png`).

Исходный режим запуска сохранён: `npm start` всё так же поднимает сервер на http://localhost:3000.

---

## 1. Подготовка (один раз)

Нужен **Node.js 18+** и открытый интернет (сборщик докачивает рантайм Electron).

```bash
npm install
```

> Важно: ставить нужно обычным `npm install` (без `--ignore-scripts`), иначе не скачается бинарник Electron.

## 2. Запуск в режиме разработки (окно сразу, без сборки)

```bash
npm run electron
```

Откроется окно с заставкой → затем приложение. Данные пишутся в системную папку профиля приложения.

## 3. Сборка установщика `.exe` (Windows)

```bash
npm run dist:win
```

Результат появится в папке `dist/`:

```
dist/GeoRail-Setup-1.0.0.exe   ← это и есть установщик для пользователей
dist/latest.yml                ← манифест для авто-обновлений (нужен при публикации)
dist/GeoRail-Setup-1.0.0.exe.blockmap
```

Установщик однокликовый (`oneClick`): ставится в профиль пользователя **без прав администратора**,
создаёт ярлык на рабочем столе и в меню «Пуск», после установки сразу запускает приложение.

> Установщик Windows надёжнее всего собирать **на Windows**. На Linux/macOS это тоже возможно, но
> потребуется `wine` — проще собрать на Windows-машине.

## 4. Авто-обновления через GitHub Releases

Приложение при запуске проверяет наличие новой версии и, если она есть, скачивает её в фоне и
предлагает перезапуститься. Чтобы это заработало:

1. Создай **публичный** репозиторий на GitHub, например `georail-schedule`.
2. В `package.json` в блоке `build.publish` замени `ЗАМЕНИ_НА_СВОЙ_GITHUB_ЛОГИН` на свой логин
   (и при необходимости `repo`).
3. Сгенерируй Personal Access Token (scope `repo`) и положи его в переменную окружения:
   - Windows PowerShell: `$env:GH_TOKEN="ghp_..."`
   - bash: `export GH_TOKEN=ghp_...`
4. Опубликуй релиз:
   ```bash
   npm run release
   ```
   Команда соберёт `.exe` и зальёт его вместе с `latest.yml` в GitHub Release с тегом `v1.0.0`.

### Как выпускать следующие версии

1. Подними версию в `package.json` (например, `1.0.1`).
2. Снова `npm run release`.
3. У пользователей при следующем запуске появится окно «Доступно обновление» → «Перезапустить сейчас».

> Если не хочешь автоматическую публикацию — можно собрать `npm run dist:win` и вручную загрузить
> файлы из `dist/` (`.exe`, `latest.yml`, `.blockmap`) в GitHub Release с тегом `v<версия>`.
> Главное — чтобы в релизе лежал `latest.yml`, иначе авто-обновление не увидит новую версию.

## 5. Где лежат данные пользователя

- Windows: `%APPDATA%\GeoRail\data`
- Открыть быстро: меню **Справка → Папка с данными**.

Данные не удаляются при обновлении приложения.

---

## 6. Подпись кода (опционально)

Неподписанный `.exe` вызывает предупреждение SmartScreen при первом запуске («Windows защитила ваш компьютер»).
Пользователь может нажать «Подробнее → Выполнить в любом случае». Чтобы убрать предупреждение, нужен
сертификат для подписи кода (Code Signing Certificate) и настройка `win.certificateFile`/`certificatePassword`
в `electron-builder`. Это платно и необязательно для внутреннего использования.
